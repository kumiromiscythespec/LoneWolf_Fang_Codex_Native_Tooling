# Code of Conduct

## Purpose

This project expects respectful, safety-minded collaboration. The workflow is
owner-gated because bypassing review can create security, privacy, runtime, or
publicization risk.

## Expected Behavior

- Treat contributors, reviewers, and the owner with respect.
- Welcome careful safety review and clear disagreement.
- Keep feedback specific, actionable, and tied to the approved scope.
- Preserve explicit owner gates for commit, push, publicization, application
  submission, notification work, runtime work, and private API work.
- Report sensitive concerns without dumping secrets or private payloads.

## Unacceptable Behavior

- Harassment, threats, or abusive comments.
- Pressure to bypass owner approval gates.
- Posting secrets, raw private logs, raw conversations, credentials, contact
  details, or private API payloads.
- Encouraging deploy, runtime, trading, notification, publicization, or
  application submission without explicit current approval.

## Reporting

Reporting and enforcement contact: owner-defined / pending publicization review.

If the safe reporting path is unclear, stop with:
`Stop and Wait - Owner Review Required.`

This document is a lightweight collaboration guide. It does not overclaim legal
enforcement authority.
