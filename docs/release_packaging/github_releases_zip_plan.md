# GitHub Releases ZIP Plan

This document records a future final-stage packaging plan for the public Codex
Native Closed Loop template.

## Final-stage Requirement

At the final finishing stage, public users should be able to download the
public template file set as one ZIP from GitHub Releases.

Recommended first version:

```text
v0.1.0-alpha.1
```

Recommended release asset name:

```text
codex-native-closed-loop-v0.1.0-alpha.1.zip
```

The release ZIP is a public template bundle. It is not a private project
artifact, not an AppData handoff packet, not a runtime package, and not a
private project profile export.

## Approval Boundaries

This plan does not create a release ZIP. No release ZIP exists yet.

This phase does not create a Git tag. No tag exists from this phase.

This phase does not create a GitHub Release. No GitHub Release exists from this
phase.

This phase does not upload a release asset. No release asset is uploaded from
this phase.

This phase does not publicize the repository. Public visibility change remains
separate.

This phase does not submit an OpenAI application. OpenAI application submission
remains separate.

Future ZIP dry-run is separate. Future tag creation is separate. Future GitHub
Release creation is separate. Future release asset upload is separate. Future
public visibility change is separate.

Do not write docs as if the release is already available. Do not provide a
download link until a future owner-approved release asset exists.
