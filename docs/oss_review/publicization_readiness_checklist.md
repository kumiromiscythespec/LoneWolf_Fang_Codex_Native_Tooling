# Publicization Readiness Checklist

Publicization is not approved.

Before any owner manual GitHub UI publicization decision, review:

- LICENSE present and intended by the owner;
- SECURITY.md present;
- CONTRIBUTING.md present;
- code of conduct decision recorded;
- private content scan complete;
- secret scan complete;
- external artifact ZIPs excluded from the repo;
- raw logs excluded;
- raw ChatGPT/Codex dumps excluded;
- provider credentials excluded;
- notification tokens excluded;
- private repo references reviewed;
- owner-local paths excluded from public setup defaults;
- personal username paths excluded;
- private project hardcoding excluded;
- hardcoded artifact roots excluded;
- public template docs separated from private project profiles;
- Public Radar boundary documented;
- Release ZIP plan documented;
- Release ZIP exclusion policy documented;
- Release ZIP validation checklist documented;
- GitHub Release creation remains separate approval;
- release asset upload remains separate approval;
- no AppData artifacts in release bundle;
- no owner-local required defaults in release bundle;
- no deployment or publicization automation;
- final owner review complete.

The repository must remain PRIVATE until the owner makes a separate future
publicization decision.
